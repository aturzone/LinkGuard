LinkGuard Privacy Policy

Last updated: February 19, 2026

Overview

LinkGuard is a browser extension that checks URL safety using offline heuristic analysis. We are committed to protecting your privacy. This policy explains what data LinkGuard handles and how.

Data Collection

LinkGuard does NOT collect, transmit, or store any personal data on external servers.

Specifically, LinkGuard:

  • Does NOT send URLs or browsing data to any external server
  • Does NOT use analytics or tracking services
  • Does NOT require user accounts or registration
  • Does NOT make any network requests for URL analysis
  • Does NOT share data with third parties

Local Data Storage

LinkGuard stores the following data locally in your browser using the browser's built-in storage API (chrome.storage.local):

  • Configuration settings: Your preferences including warning/block thresholds, intercept mode, notification level, allowlist, and blocklist domains.
  • Scan history: A log of recently analyzed URLs with their safety scores (up to 1,000 entries). This data never leaves your browser.

You can clear all stored data at any time through the extension's popup interface.

How URL Analysis Works

All URL analysis is performed entirely within your browser using local heuristic algorithms. The extension examines:

  • Domain structure and reputation (against a bundled safe-domains list)
  • URL structure patterns (length, encoding, special characters)
  • Protocol safety (HTTPS vs HTTP)
  • Content patterns (phishing keywords, brand impersonation)
  • Behavioral patterns (redirects, entropy analysis)

No part of this analysis involves external API calls or network requests.

Permissions

LinkGuard requests the following browser permissions:

  • activeTab: To analyze the URL of the currently active tab
  • storage: To save your configuration and scan history locally
  • webNavigation: To intercept navigation and check URLs before loading
  • notifications: To alert you about dangerous links (configurable)
  • host_permissions (<all_urls>): Required to analyze links on all websites you visit

Configuration Export/Import

The config export feature creates a JSON file on your local device. The import feature reads a JSON file from your local device. No configuration data is transmitted over the network.
